# Error

`kasane_logic` のエラーは、発生源ごとに階層化して整理している。

## 概要

最上位の [`Error`](../src/error.rs) は、公開 API が返す共通の入口である。
内部では大きく次の 2 系統に分けている。

- Geometry 系
- SpatialId 系

利用者は通常、まず `Error` を受け取り、必要に応じて内側の分類で分岐する。

## ねらい

この構成は、どの層で失敗したかを保ちながら、利用者側の受け口を 1 つに保つためのものである。
細かな個別エラーを文書化しなくても、設計の境界だけ理解できれば十分である。

## 使い分けの感覚

- Geometry 系は、座標や図形の妥当性に関わる失敗を表す。
- SpatialId 系は、ズームレベルやインデックス、時間区間などの空間 ID 生成・変換に関わる失敗を表す。

## 補足
