# FlexTree の Set / Table の違い

このドキュメントは、`SpatialIdSet`、`SpatialIdTable` の違いを、保持する値と役割に絞って整理したものである。

## `SpatialIdSet`

`SpatialIdSet` は、空間 ID の集合を表す型である。

- 保持する値: 空間 ID そのものだけである。値に相当する要素は持たない。
- 役割: 「どの空間が存在するか」を表すための型である。
- 向いている用途: 領域の有無を判定したい場合や、重なり・和・差・共通部分のような集合演算を行いたい場合である。

## `SpatialIdTable`

`SpatialIdTable` は、空間 ID に値を対応付けつつ、値と空間 ID の対応関係をまとめて管理する型である。

- 保持する値: 空間 ID と値の対応に加えて、値ごとの索引も内部に保持する。
- 役割: 「この空間には何が入っているか」と「同じ値を持つ空間群をまとめて扱う」ことの両方を担う型である。
- 向いている用途: 空間ごとに属性や状態を持たせたい場合に加えて、値からの検索、値の範囲検索、値ごとの空間の集約を行いたい場合である。

## 使い分け

- 位置だけが重要なら `SpatialIdSet` を使う。
- 空間ごとに値を持たせたい、値から空間を引きたい、または値の一覧管理が必要なら `SpatialIdTable` を使う。

## 補足

`SpatialIdSet` は空間の集合、`SpatialIdTable` は空間に値を載せつつ値を軸にした逆引きも備える対応表である。空間ごとに 1 つの値を持たせたいだけの用途も `SpatialIdTable` が担う。値型には [`Ord`] が要求される（内部の索引で値をキーに使うため）。
